# Agentic AI "Zarigani" の記憶層の実装計画

## 1. 利用データセットのデータ型とデシリアライズを実装（完了）

利用するデータセット

- LoCoMo
- LongMemEval

## 2. 記憶層の呼び出し I/F を決める

前提: Agentic AI とのチャットは Slack/Discord/Google Chat/Telegram/WhatsApp などのチャットサービス上で行われることを想定する。

それらのチャットの会話を分割する単位を抽象化する。

1. **Space**: Slack/Discord/Google Chat などのサービスで、さらにそのサービス内の「特定の集団専用」の単位（Slack ならワークスペース、Discord ならサーバー）
2. **Room**: 話題（トピック）ごとに別々に用意されている会話ルーム的なもの。 Discord ならチャネル。
3. **Thread**: ある発言に対する返信として連続する会話スレッド。スレッド内の返信にさらにスレッド分岐できたりするサービスもある。従ってスレッドは階層構造を持つ。

記憶層の呼び出しは、以下の２パターンがある。

- Agentic AI の応答を返すために LLM を呼び出すために記憶を検索する場合
- Agentic AI 自身の応答を記憶層に記憶させるために呼び出す場合

後者の場合は記憶層からの応答は必要ない。

- 記憶の呼び出し関数
    - 入力
        - Space ID
        - Room ID
        - Thread IDs
        - User ID （発言者 ID）
        - User Name （発言者名）
        - 発言内容
        - 発言のタイムスタンプ
        - メタデータ（記憶層では利用せず、単にログに保存）
        - 記憶層からの応答を取得するかどうかのフラグ
    - 出力
        - Optional: 記憶層からの応答（LLM に渡すプロンプトに挿入する記憶部分）

ベンチマーク用に、記憶層がバックグラウンドで動いている状態をシミュレーションする仕組みがいる。
つまり次の入力のタイムスタンプを実時間だけ待つのは非現実的なので、次の入力のタイムスタンプよりも前に全てのバックグラウンド処理が終わったら、次の入力を処理するような仕組みが必要。

- コールバック関数の設定
    - コールバック関数の引数
        - なし
    - コールバック関数の戻り値
        - なし

コールバック関数が呼ばれたら、次のメッセージのタイムスタンプが実時間ではまだ入力タイミングになっていなくても、次のメッセージを処理する。

## 3. 記憶層のスタブを作る

渡されたメッセージを溜め込んで、記憶として全てそのまま返すスタブを作る。

## 4. 各種ベンチマークデータセットを利用した評価プログラムを作る

ベンチマーク用にコールバック関数を用意したものの、実際のところ LoCoMo も LongMemEval も、各発言のタイムスタンプはない。
そのため擬似的に、前の発言から一定時間後に次の発言が行われるという前提で、処理を進めていく。
